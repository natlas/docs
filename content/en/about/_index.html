Continuous Recon For The Rest Of Us

{{< /blocks/cover >}} {{% blocks/lead %}} Natlas is an open source continuous reconnaissance platform which prioritizes identifying exposure, rather than identifying vulnerabilities. Identifying vulnerabilities often contains many similar steps that Natlas uses, however Natlas emphasizes collecting the most fundamental level of data, upon which further insight can be drawn. {{% /blocks/lead %}} {{< blocks/section >}}

Open Source

Open source is at the core of the Natlas project. Natlas makes extensive use of existing open source tools and libraries, and it's only fair that we pass that along with our project. Natlas and it's associated projects are licensed under Apache 2.0 unless otherwise stated, which means you're free to use, distribute and modify Natlas to your hearts content. Just fork the repository you're interested in working on and jump right in!
{{< /blocks/section >}} {{< blocks/section >}}

Continuous Reconnaissance

What does "continuous reconnaissance" mean? Well quite simply, it means that Natlas never stops scanning. When it completes it's scanning cycle, it starts again automatically. Port exposure is something that can change regularly and without notice, so it is unreliable to reference old port scans when making decisions. There's no licensing based on the number of addresses scanned, no fees, no artificial restrictions. You can scan as many addresses as fast as your deployment will let you. And if that's not fast enough, you can just scale up.
